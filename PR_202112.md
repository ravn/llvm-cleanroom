Fixes #202112.

## Summary

This fix allows `TruncInstCombine` to also consider input parameters as valid sources. 

This showed up on a K&R source on a Z80, where an 8-bit input parameter is widened to 16 bits at the ABI boundary, giving expensive 16-bit arithmetic where 8-bit would do.  It also applies to the AVR in-tree.

The fix is to treat `Argument` as a leaf in the expression graph, parallel to the existing `Constant` case. 

The PR includes a new lit test that covers the original K&R case and some additional cases.

I have done this using Claude, but have tried to do this as clear and simple from the issue as I could. 

The text below explains things in more detail as written by Claude.  I have reviewed it but not rewritten it, as it was clear enough on its own.

## Detailed analysis (by Claude `claude-opus-4-7`, reviewed by Thorbjørn)

`TruncInstCombine::buildTruncExpressionGraph` accepted `Constant` and
`Instruction` leaves but bailed on anything else, so expressions whose
root operand is a function parameter (`Argument`) couldn't be narrowed.
The rejection happens during graph construction, before the
min-bit-width analysis ever runs — it's not a soundness conclusion,
just a missing case.

The same expression shape arrives via `zext` (with an ANSI prototype) or
via `load` today and is narrowed correctly; only the `Argument` leaf
was missing.

### Before / after on the reduction from the bug

Before (no narrowing — the entire graph stays at i16):

```llvm
define zeroext i8 @rotl(i16 noundef %x) {
  %m = and i16 %x, 255
  %s = shl nuw nsw i16 %m, 1
  %r = lshr i16 %m, 7
  %o = or disjoint i16 %s, %r
  %t = trunc i16 %o to i8
  ret i8 %t
}
```

After this patch (with downstream `instcombine` cleanup eliminating the
`and X, -1` residue):

```llvm
define zeroext i8 @rotl(i16 noundef %x) {
  %x.trunc = trunc i16 %x to i8
  %s = shl i8 %x.trunc, 1
  %r = lshr i8 %x.trunc, 7
  %o = or i8 %s, %r
  ret i8 %o
}
```

### Design

Four small changes, all in `TruncInstCombine.cpp`:

1. **`buildTruncExpressionGraph`** — admit `Argument` as a leaf, parallel
   to the existing `Constant` case.
2. **`getMinBitWidth`** — extend the `Src` short-circuit to also handle
   `Argument`. Mirrors the existing `Constant` early-return; without it,
   the `cast<Instruction>(Src)` two lines later would crash.
3. **`getBestTruncatedType`** — skip the trivial `trunc Arg to T` case
   where the rewrite would only relocate the trunc to function entry
   with no real benefit. (Constant leaves still benefit because the
   trunc folds away to a constant.)
4. **`getReducedOperand`** — materialise `trunc <arg> to <narrow-ty>`
   once at function entry via `SetInsertPointPastAllocas`. Per-Argument
   deduplication across in-graph uses is left to follow-on `InstCombine`,
   consistent with how the pass already relies on cleanup for `and X, -1`
   residue.

### Why it's sound

An `Argument` carries no width information of its own, so the existing
min-bit-width analysis (driven entirely by in-graph instructions: shift
amounts, `lshr`/`ashr` `KnownBits`, `udiv`/`urem` operand widths) is
not weakened by treating `Argument` as a width-free leaf. The same
equivalence that makes `Constant` leaves safe applies:
`trunc(graph(args, consts))` ≡ `graph(trunc(args), trunc(consts))`
modulo `2^NarrowBits`, for every binary op the pass already accepts.

### Testing

`llvm/test/Transforms/AggressiveInstCombine/trunc_argument.ll` adds:
- the K&R `rotl` reduction from the bug
- a longer xor/shl/lshr/or chain rooted at an `Argument`
- a trivial `trunc Arg → T` (must remain a no-op)
- a too-large shift-amount negative
- a no-leading-mask negative (the `KnownBits` shift-LHS gate must
  still bind)

Each positive case fails on unpatched `opt` and passes with the patch
applied.

Lit results:
- `Transforms/AggressiveInstCombine` — 50/50 pass (3 are target-conditional
  `Unsupported`, same as the baseline).
- Broader sweep over `Transforms/{InstCombine,AggressiveInstCombine,PhaseOrdering}` —
  1991 of 2078 pass, 86 `Unsupported`, 1 `XFAIL`, **0 regressions** from
  this change.
